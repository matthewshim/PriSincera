## 1. 💡 導入：哲学は一つ、なのにメニューは二つだった

PriSinceraの成長哲学は、最初から一つの文でした — **学びが実行になり、実行が振り返りを経て次の学びを個人化する。** 毎日の知識キュレーション(Daily Digest)と週次の実行・振り返りトラッカー(Pace Note)は、このフライホイールの前輪と後輪として設計されたサービスでした。

問題は、ユーザーにとってこの二つが**完全に分離した二つのメニュー**に見えていたことです。ダイジェストで良い記事を読んでも、それを実行へ移す動線がなく、軌道(Orbit)を完走しても、その経験が翌日の学びに反映されるという事実を画面のどこでも実感できませんでした。

皮肉なことに、**バックエンドはすでに繋がっていました。** 成長ループのパイプライン(Growth Loop Phase 0〜4)が、完了・振り返りシグナルを推薦エンジンとダイジェストのレンズへ還流していたのです。残されたのはただ一つ — **この見えない繋がりを画面上へ引き上げる、最後の表層作業**でした。

> 💡 **一言で**：統合の本質は新機能の開発ではなく、すでに存在していたデータのフライホイールに「一つの旅」というUIをまとわせることでした。

---

## 2. 🏷️ 名づけ：Re-Learn、そしてRe-run

統合サービスの名は **ReLearn(リラーン)** に確定しました。「**再び学ぶ(Re-Learn)**」と「**再び走る(Re-run・ランナー)**」の二重の意味を一語に込め、サービスのターゲットである*「ゼロから学び直すラーナー(Learner & Runner)」*を、名前そのものが象徴するようにしました。

> **タグライン**：「毎日ゼロから、また学び、また走る — Learn from zero, run again.」

ZeroRun・Day One・Base Campといった候補も検討しましたが、「統合」という出来事そのものを名前が説明できるのはReLearnだけでした。

---

## 3. 🧭 戦略的決断：併存ではなく、継承(Successor)

最も重要な意思決定は技術ではなく、**ポジショニング**でした。

当初は既存の二サービスをそのまま残し、ReLearnを並列追加する「追加型(Additive)」戦略から出発しました。しかし計画を練る中で、これを再定義しました — **ReLearnは二サービスの継承者(Successor)であり、「追加型」は永続的な併存ではなく、移行期の非破壊的な移行戦略である。**

この再定義から生まれた三つの原則：

1. **パリティゲートなしにサンセットなし** — 購読解除の内製化、アーカイブ閲覧、カレンダー・オムニ検索の廃止/移管判定まで、機能パリティチェックリスト(P1〜P6)をすべて解消した後にのみ、既存メニューを下ろす。
2. **露出終了 ≠ URL削除** — `/daily/:date`のようなコンテンツ詳細URLは、数か月かけて積み上げたSEO資産です。メニューから下ろしても、URLは永続保存し、旧経路はサーバー301で新アドレスへ継承します。
3. **切り替えは検証後、最後に** — 新ルート`/relearn`をまず開き、安定を確認してから初めて入口を差し替えます。

## 4. 🛠️ アーキテクチャ：新規API 0、新規ジョブ 0

この規模の統合プロジェクトで最も自慢したい数字は、華やかな新機能ではなく**0**です。

### 1) 方向の分離 — パイプラインに無接触

バッチパイプライン(Cloud Run Jobs)はコンテンツを**生成(書き込み)**する経路であり、ReLearnは既存APIを**読み**、既存のユーザー書き込みAPIを再利用するだけです。新規API 0件、新規ジョブ 0件、`cloudbuild.yaml`の変更 0件。ReLearnのビルドが失敗しても、早朝のパイプラインは既存イメージで正常に稼働します。

### 2) 推薦統計の汚染防止

ReLearnの核となるUX「軌道へ」ワンクリックは、カスタム軌道(`custom-`)を大量に生成しうります。これが推薦プールのvelocity・淘汰統計を歪めないよう、パイプラインの統計集計からカスタム軌道を明示的に除外しました。

```javascript
// pacenote-composer — ReLearn発のカスタム軌道は推薦プール統計から除外
const poolTasks = weekTasks.filter(task => !task.id.startsWith('custom-'));
```

### 3) モノリスの解体 — 抽出こそが負債返済

学びのカードは旧DailyDigestページに、実行・振り返りは約1,900行のPaceNoteDashboardに、それぞれ一塊で束ねられていました。ReLearnのシェルへ組み立てるにはまず**セクションコンポーネントへ抽出(Phase B-0)**する必要があり、ここで一つの原則を立てました — **抽出したコンポーネントは、既存ページも一緒に消費するよう差し替える。** コピーではなく一元化なので、統合作業そのものがコード負債を減らす方向に働きました。

### 4) 重複フェッチの禁止

シェルが学び・実行のステージを一緒に描くため、プロフィール・軌道・トラックフィードは**シェルレベルで一度だけフェッチ**し、下位へ渡します。ステージごとに各自呼び出した瞬間、同じAPIが画面あたり3回叩かれる構造になるからです。

## 5. 🔄 一つの旅：今日 | 記録、そして3-Stageループ

ReLearnの画面は、単一ルートの中で **`今日 | 記録`** の二つのビューを切り替えます。

```
┌─ ReLearn (今日) ─────────────────────────────────────┐
│  ─ 成長ループレポート (完了率 · 連続 · 集中ドメイン) ─   │
│  ① 学び   ダイジェスト4チャネル(シグナル·トラック·プロンプト·語学) │
│  ② 実行   ワンクリック「実行の軌道へ追加」→ 今日のタスク  │
│  ③ 振返   完了チェック + 一行の振り返り                │
└──────────────────────────────────────────────────────┘
```

- **学びの4チャネルすべて**が軌道と繋がります。テックトラックのアクションチャレンジは既存の`add-orbit`で、残り3チャネル(シグナル・AIプロンプト・語学)は既存のカスタム軌道APIの再利用で — こちらも新規API 0です。
- **ループレポートは両ビューに常駐**する橋です。完了率・連続・集中ドメインのタイルを押すと、記録ビューの該当週へドリルダウンします。
- 認証状態は、未ログイン / コールドスタート / 軌道あり / 今週未開始 の**4状態マトリクス**へ精密に分岐し、各状態で次の行動が明確な画面を見せます。

面白い発見もありました。Pace Noteのデフォルト軌道5種(「AIスタディプロンプトを自分で実行してみる」「ビジネス日本語の文を3回読む」…)は、実はこのチャネル-軌道マッピングの**ハードコード版**でした。大統合はこれを「今日の実際のコンテンツから動的に追加」へ昇格させ、デフォルト軌道はコールドスタートのフォールバックとして残りました。

## 6. 🌅 継承の実行：3段階のサンセット

パリティゲートが解消された後、継承は3段階で実行しました。

| Phase | 内容 |
| :--- | :--- |
| **Phase 1** | ReLearn独自のアーカイブ詳細(`/relearn/daily/:date`)を構築 — 過去日付閲覧の最後の依存を取り除き、内部リンクをすべて新経路へ切り替え |
| **Phase 2** | 入口の差し替え — GNB・ホーム・メールの入口をReLearnへ移し、旧経路(`/daily`, `/pacenote`)は**サーバー301**でSEOを移管 |
| **Phase 3** | 旧ページのコード・ルート・メタ・ドキュメント位相を整理 — 二つの大型ページコンポーネントがリポジトリから退役 |

周辺の整合も併せて終えました。ホームのサービスセクションはDaily・Paceの二枚のカードをReLearnフラッグシップカード一枚へ統合し、AdminコンソールはReLearn体制に合わせてグループ4→3・タブ10→8へ再編、毎日配信のニュースレターは「📬 ReLearn Daily」へブランディングを継承しました。

## 7. 🎯 結び：統合は引き算だった

この大統合で学んだことを一文に縮めるなら — **良い統合は足し算ではなく引き算だ。**

- 新規API・新規ジョブ・新規カラートークン、すべて**0**
- モノリシックな2ページ → 再利用可能なセクションコンポーネントへ解体(負債返済)
- メニュー2つ → 旅1つ、それでもURLとSEO資産は100%保存
- バックエンドにすでに存在していた成長フライホイールが、ついに画面で見えるように

PriSinceraのプロダクトポートフォリオは、いまや **ReLearn(成長ループ) · Builder's Log(制作記) · Sylphio(オンデバイス通訳)** の3プロダクト体制です。毎日ゼロから、また学び、また走る。🏃
