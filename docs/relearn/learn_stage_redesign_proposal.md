## 0. 요약 (TL;DR)

- STAGE 01 배움은 **/daily의 상세 뷰를 거의 그대로 4채널 연속 스택**한 상태라, 한 화면 분량이 아니라 **"오늘치 신문 전체를 다 펼쳐놓은"** 구조입니다. 특히 트랙 채널은 **카드 수 상한이 없고** 카드마다 학습·챌린지 블록이 전부 펼쳐져 지면 소비가 가장 큽니다.
- 4채널이 **동일한 섹션 헤더·동일한 배경**을 쓰기 때문에 채널 경계가 시각적으로 존재하지 않습니다. 좌측 책갈피 하이라이트만으로는 전환 인지가 어렵습니다.
- 처방: **① Compact-first(요약 우선) + Progressive Disclosure(펼침은 선택)**, **② 채널 존(Zoning) 컬러 시스템**, **③ 항목별 생략/축소**. ReLearn의 배움은 "정독 아카이브(/daily의 역할)"가 아니라 **"오늘의 배움 → 궤도 추가"라는 루프 진입 관문**이므로, 요약 밀도를 높이고 상세는 /daily·확장으로 위임하는 것이 정보 구조상으로도 옳습니다.

| 🛰️ 트랙 | 트랙 토글 + 도메인 칩 + **카드 N개(상한 없음)** × [도메인·태그 5개 + 제목 + 요약 + 📚학습(개념+포인트 리스트) + 🎯챌린지(제목+할일 3개+궤도 버튼) + 원문 링크] | 카드당 ~400px × N | **최대 지면 소비원.** 카드 하나가 미니 아티클 수준 |

## 2. 리디자인 원칙

1. **Compact-first, Progressive Disclosure** — 기본 화면은 "오늘 무엇을 배울지 고르는 메뉴판". 상세(학습 포인트·챌린지 할일·파라미터·어휘)는 **펼침/이동 시에만**.
2. **채널 = 존(Zone)** — 채널마다 고유 액센트 컬러의 패널 존으로 감싸 "지금 어느 채널인가"를 배경 레벨에서 인지. 좌측 책갈피 하이라이트 색과 동기화.
3. **Loop-first** — 배움의 성공 지표는 정독 완료가 아니라 **궤도 추가(orbit_add)**. 모든 축약은 이 전환을 해치지 않는 방향으로.
4. **원본 보존** — SignalSection 등은 /daily와 공용이므로, 축약은 **prop(`compact`)으로 분기**해 /daily 기존 화면은 불변.

## 3. 제안 ① — 정보 생략/축소 검토표 (항목별)

| 채널 | 항목 | 처분 | 근거 |
| :--- | :--- | :--- | :--- |
| 트랙 | 카드 수 | **상한 3개** + "트랙 전체 보기" 링크 | 시그널 limit=4와 동일한 원칙. 개인화 정렬 상위 3개면 루프 진입에 충분 |
| 트랙 | 태그 | 5개 → **2개** | 식별 목적엔 2개로 충분, 메타 행 줄바꿈 방지 |
| 트랙 | 📚학습 블록 | **접기(요약 1줄 → 펼침)** | 카드 최대 지면원. 개념 1줄 프리뷰로 호기심 유도 |
| 트랙 | 🎯챌린지 할일 3개 | **접기** — 제목+궤도 버튼만 기본 노출 | 할일 상세는 궤도 추가 후 STAGE 02에서 어차피 재노출됨(중복) |
| 시그널 | 커버 이미지 | **compact 모드에서 제거**(또는 좌측 56px 썸네일) | 배움 관문에서 이미지는 장식 — 텍스트 우선. /daily는 유지 |
| 시그널 | 카드 레이아웃 | 2열 대형 카드 → **1열 콤팩트 리스트 행**(카테고리·제목·인사이트 1줄) | 지면 ~1/3로 축소, 스캔성 상승 |
| 시그널 | 개수 | 4개 유지 | 이미 상한 있음 |
| 프롬프트 | 스니펫 | **6줄 프리뷰 + 펼치기** (복사 버튼은 항상 노출) | 복사가 핵심 행동 — 전문 정독은 선택 |
| 프롬프트 | 파라미터 테이블 | **접기** | 파라미터는 복사 후 편집 시점에 필요한 참고 정보 |
| 어학 | 문장·발음·번역 | **유지** (채널의 본질) | — |
| 어학 | 컨텍스트 팁 | 유지 | 짧고 가치 높음 |
| 어학 | 어휘 그리드 | **4개 상한 + 펼치기** | 전체 나열은 /daily 몫 |
| 공통 | 채널 하단 "궤도 추가" 버튼 | 유지하되 **문법 통일**: 트랙 외 3채널도 카드/블록 우측 상단의 경량 아이콘 버튼으로 통일 검토 | P4 해소. 단, 구현 2단계로 분리 가능 |

## 4. 제안 ② — 채널 존(Zoning) 컬러 시스템

디자인 시스템(§1-3 Celestial Accents)의 기존 토큰만 사용해 채널 아이덴티티를 부여합니다.

| 채널 | 액센트 | 존 배경(틴트) | 비고 |
| :--- | :--- | :--- | :--- |
| 🛰️ 트랙 | `--color-cyan` | `rgba(34,211,238,0.035)` | 테크 시그널 = 시안(기존 링크·테크 의미) |
| 📰 시그널 | `--prism-lavender` | `rgba(199,210,254,0.035)` | 뉴스·정보 = 라벤더 |
| 🤖 프롬프트 | `--color-indigo` | `rgba(165,180,252,0.035)` | AI = 인디고 |
| 🇯🇵 어학 | `--color-gold` | `rgba(229,178,93,0.035)` | 어학 원픽 = 골드 |

**존 패널 규격**
- 각 채널 섹션을 `radius-lg` 글래스 패널로 감싸고: 배경 틴트 + **좌측 3px 채널 컬러 보더** + 상단 **채널 미니 헤더**(아이콘+채널명+채널 컬러 언더라인).
- 존 사이 간격 34px → **56px + 페이드 디바이더**로 확대 — "장(章)이 바뀐다"는 리듬 부여.
- **좌측 2뎁스 책갈피 활성색을 채널 컬러와 동기화** (현재는 전부 시안) — 스크롤스파이 하이라이트와 존 배경색이 함께 바뀌므로 전환 인지가 이중으로 보강됩니다.
- 스크롤스파이 기준선(260px)과 존 상단 경계 정합 유지.
