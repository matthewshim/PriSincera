# PriSincera 메인 페이지: 통합 3D/WebGL 리뉴얼 기획안

기존 히어로 영역에 국한되었던 개편 논의를 확장하여, **PriSincera 메인 페이지(Home) 전체를 하나의 거대한 3D 우주 공간으로 통합**하는 마스터플랜입니다. 사용자가 스크롤을 내릴 때마다 끊어지는 배경이 아닌, 카메라가 우주를 비행하며 유리 패널 형태의 콘텐츠들을 만나는 **심리스(Seamless) 스크롤리텔링(Scrollytelling)** 경험을 제공합니다.

---

## 1. 아키텍처 패러다임 전환 (Global 3D Canvas)

* **현재 구조:** `<section>` 단위로 배경색과 콘텐츠가 분할되어 있으며, 각 섹션이 독립적인 2D 스크롤을 가집니다.
* **리뉴얼 구조:** 화면 최하단에 **고정된(Fixed) 단일 WebGL `<canvas>`**를 배치하여 전역 우주 배경을 렌더링합니다. HTML DOM 요소들(텍스트, 카드, 버튼)은 그 위를 떠다니는 투명한 레이어(Overlay)로 작동하며, DOM의 스크롤 위치에 따라 3D 카메라의 Z축(깊이)이 동기화되어 전진합니다.

---

## 2. 섹션별 시각화 및 인터랙션 시나리오

### A. Hero Section (시작점: 우주의 중심)
* **비주얼:** 수만 개의 별이 반짝이는 한가운데, 브랜드 철학을 상징하는 **3D 크리스탈 프리즘(Star Constellation 로고)**이 천천히 자전합니다.
* **인터랙션:** 마우스를 움직이면 프리즘이 빛을 굴절(Refraction)시키며 그림자가 왜곡됩니다. 커서를 가까이 대면 주변의 별들이 자력에 의해 커서 주변으로 은하수처럼 모여듭니다 (Magnetic Swarm).

### B. Philosophy Section (전진: 이념의 공간)
* **스크롤 트랜지션:** 마우스를 스크롤하는 순간, 카메라가 프리즘을 꿰뚫고 지나가며 우주 속으로 빠르게 **줌인(Zoom-in)** 합니다. (워프 효과)
* **비주얼:** 텅 빈 공간 속에서 투명한 유리 재질(Glassmorphism 3.0)의 텍스트 패널들이 부유하듯 떠오릅니다.
* **인터랙션:** 패널 뒷면으로 배경의 별들이 지나갈 때, 유리에 비친 빛이 왜곡되고 산란되어 極상의 고급스러움을 연출합니다.

### C. Work Section (발견: 궤도 위의 프로젝트들)
* **비주얼:** 각 프로젝트 카드들이 우주 공간의 궤도(Orbit)를 따라 정렬된 행성계처럼 배치됩니다. 카드 자체는 초고품질의 **두꺼운 프로스트 글래스(Frosted Glass)** 효과를 가집니다.
* **인터랙션:** 카드를 호버(Hover)하면 카드가 스프링처럼 쫀득하게 튀어 오르며, WebGL 배경의 별자리 에너지 라인들이 해당 카드 쪽으로 쏠리는 '환경 반응형' 애니메이션이 연출됩니다.

### D. Journey Section (기록: 별의 궤적)
* **비주얼:** 기존의 수직/수평 2D 타임라인 선을 폐기하고, 3D 공간을 가로지르는 **거대한 빛의 궤적(Light Beam)**으로 표현합니다. 
* **인터랙션:** 각 연도별 마일스톤(Node)에 스크롤이 도달할 때마다 해당 노드가 초신성처럼 폭발(Bloom)하며 데이터를 화면에 투사합니다.

### E. Connect Section (종착점: 중력의 우물)
* **비주얼:** 스크롤의 가장 끝에 도달하면, 강한 에너지를 뿜어내는 **블랙홀 또는 태양광 코어(Core)**가 등장하여 사용자의 시선을 집중시킵니다.
* **인터랙션:** 마우스 커서를 올리면 코어가 마우스를 강하게 끌어당기는 듯한 거대한 중력 왜곡 효과를 주어, 문의하기(CTA) 버튼 클릭을 압도적으로 유도합니다.

## 3. 통합 물리 엔진 (Unified Physics)
메인 페이지의 모든 요소(HTML DOM & WebGL Object)가 **단일한 스프링 물리 법칙(React Spring / Framer Motion 3D)** 아래 통제됩니다.
* 마우스 이동 시 관성(Mass)과 마찰(Friction) 적용.
* 버튼, 카드 등 모든 UI 호버 시 젤리나 고무줄처럼 탄성 있는 묵직한 조작감 제공.
* 시선(카메라) 이동 시 부드러운 감속(Damping) 처리로 멀미(Motion Sickness) 원천 방지.

## 4. 극한의 퍼포먼스와 메모리 최적화 (Zero-Leak)

화려한 효과만큼이나, 브라우저가 다운되지 않고 저사양 기기에서도 부드럽게 돌아가도록 하는 **"보이지 않는 엔지니어링"**이 핵심입니다.

1. **상태 관리 배제 (No React Re-renders for Animation):**
   * 초당 60프레임으로 변하는 스크롤과 마우스 좌표를 React의 `useState`로 갱신하면 사이트가 멈춥니다. 
   * 이를 우회하여 `useRef`와 커스텀 `requestAnimationFrame` 루프를 사용, GPU로 데이터를 직행시킵니다.
2. **GPU 파티클 시스템 (InstancedMesh):**
   * 10,000개의 별과 노드들을 개별 객체로 그리지 않고, `InstancedMesh`를 통해 GPU가 **단 한 번의 요청(Draw Call 1회)**으로 처리하도록 극단적으로 최적화합니다.
3. **메모리 강제 클리어 (Explicit Garbage Collection):**
   * 사용자가 블로그나 다른 페이지로 넘어가 컴포넌트가 언마운트될 때, Three.js에 남아있는 모든 Geometry, Material, Texture를 강제로 `dispose()` 하여 메모리 누수(Memory Leak)를 0%로 차단합니다.
4. **저사양 기기 방어 (Adaptive Performance):**
   * 기기의 FPS를 실시간 모니터링하여, 프레임이 30 이하로 떨어지는 모바일 환경에서는 무거운 볼류메트릭 셰이더를 자동으로 끄고 쾌적한 2D/심플 3D 폴백(Fallback) 모드로 전환하는 로직을 탑재합니다.

## 5. 넥스트 스텝 (Next Step)
이 기획안은 웹사이트가 아니라 **"하나의 몰입형 예술 작품이자 프로덕트"**를 만드는 아키텍처입니다. 승인 시 다음과 같은 순서로 진행됩니다.
1. **의존성 셋업:** `three`, `@react-three/fiber`, `@react-three/drei`, `framer-motion-3d` 도입.
2. **Global Canvas 세팅:** 메인 페이지 구조를 뜯어고쳐 배경 Canvas와 투명 DOM 레이어 분리.
3. **Hero & Scroll Camera 프로토타이핑:** 첫 번째 우주 공간 및 스크롤 전진 효과 연출.
