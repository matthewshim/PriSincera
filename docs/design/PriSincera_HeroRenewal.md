# PriSincera Hero Section: 프리미엄 디자인 리뉴얼 기획안 (상세)

현재 PriSincera의 히어로 영역(별자리, 텔레스코프 인터랙션)은 브랜드의 철학을 담은 훌륭한 컨셉을 가지고 있으나, 2D Canvas 및 단순 SVG 애니메이션 기반으로 구현되어 있어 글로벌 최상위 수준의 압도적인 몰입감과 타격감을 주지 못하고 있습니다. 

이를 **업계 최고 수준의 프리미엄 그래픽과 인터랙션**으로 격상시키기 위한 구체적인 리뉴얼 방향성과 성능 최적화 방안을 제안합니다.

---

## 1. 구체적인 시각화 및 인터랙션 예시 (Concrete Examples)

### A. 3D Glassmorphism & Refraction (입체적 굴절 효과)
* **레퍼런스 (체감 예시):** Apple의 VisionOS UI 재질감, Vercel의 메인 페이지 3D 오브젝트.
* **구체적 구현 방안:** 
  현재의 별자리 로고(프리즘)는 2D 선(SVG)으로 그려져 있습니다. 이를 WebGL(Three.js)의 물리 기반 렌더링(`MeshPhysicalMaterial`)을 활용하여 **"두께감 있는 반투명한 크리스탈 유리 조각"**으로 구현합니다.
  마우스를 움직이면 이 3D 유리 프리즘이 서서히 회전하며, 우주 배경의 빛나는 별들이 프리즘을 통과할 때 실제 렌즈처럼 **빛이 굴절(Refraction)되고 산란(Dispersion)**되어 매우 영롱하고 깊이 있는 우주의 중심을 연출합니다.

### B. Magnetic Nodes & Spring Physics (자성 인터랙션과 스프링 물리)
* **레퍼런스 (체감 예시):** Linear.app 홈페이지의 노드망(Node-network) 애니메이션, Stripe의 구체(Globe) 인터랙션.
* **구체적 구현 방안:**
  현재는 마우스가 움직일 때 별들이 단순히 정해진 속도로 뻣뻣하게(Linear) 따라옵니다. 
  리뉴얼 후에는 스프링 물리 엔진(Tension, Friction 적용)을 도입합니다. 마우스를 빠르게 움직이면 별자리의 선들이 **고무줄처럼 탄력 있게 늘어났다가 쫀득하게 제자리로 돌아옵니다.** 
  또한, 마우스가 별자리의 꼭짓점(Node) 근처에 다가가면 **자석처럼 커서에 달라붙는(Magnetic snap)** 마이크로 인터랙션을 통해 사용자에게 확실한 손맛과 조작감을 제공합니다.

### C. Z-axis Parallax & Volumetric Light (Z축 시차와 공간 조명)
* **레퍼런스 (체감 예시):** GitHub Universe 랜딩 페이지, 최상위 WebGL 포트폴리오.
* **구체적 구현 방안:**
  스크롤을 내릴 때 화면이 위로 밀려 올라가는 2D 방식을 벗어납니다. 스크롤 시 카메라가 우주 공간의 **별자리들 사이를 관통하며 전진(Z축 Zoom-in)**합니다. 
  이때 Bloom 필터(후처리)를 적용하여, 카메라 가까이 다가오는 별은 눈부시게 빛나고, 뒤로 지나갈 때는 렌즈 플레어 효과와 아웃포커싱(Depth of Field) 효과가 극대화되어 완벽한 우주 비행의 몰입감을 줍니다.

## 2. 퍼포먼스 및 메모리 누수(Memory Leak) 방지 대책

3D 그래픽 엔진(Three.js / React Three Fiber) 도입 시 가장 경계해야 할 부분은 **GPU 과부하와 메모리 누수**입니다. PriSincera는 초경량/고성능을 지향하므로, 다음의 하드코어 최적화 기술을 기본으로 탑재합니다.

### A. 메모리 누수(Memory Leak) 원천 차단
1. **명시적인 가비지 컬렉션 (Geometry & Material Dispose):**
   * Three.js에서 생성된 3D 객체(Mesh, Geometry, Material, Texture)는 DOM에서 사라져도 브라우저 메모리(GPU)에 영구적으로 남는 특성이 있습니다.
   * 사용자가 다른 페이지로 이동하여 리액트 컴포넌트가 언마운트(`useEffect`의 return 함수)될 때, **반드시 `dispose()` 메서드를 강제로 호출**하여 GPU 메모리에서 즉시 삭제(Clear)합니다.
2. **React Re-render 우회 (useRef 활용):**
   * 매 프레임(초당 60회) 실행되는 애니메이션 값을 React State(`useState`)로 관리하면 치명적인 렌더링 병목이 발생합니다.
   * 모든 3D 애니메이션 수치는 `useRef`에 담아, 브라우저의 `requestAnimationFrame` 루프 내에서 리액트의 렌더링 사이클을 완전히 우회하여 GPU로 직접 쏴줍니다.

### B. GPU 연산 최소화 및 프레임(FPS) 방어
1. **단일 Draw Call (InstancedMesh 사용):**
   * 우주 배경에 10,000개의 별을 그릴 때 개별 객체를 만들면 CPU가 터집니다.
   * `InstancedMesh` 기법을 사용하여 **단 1번의 렌더링 요청(Draw Call)**만으로 형태가 같은 수만 개의 별을 한 번에 화면에 뿌립니다.
2. **뷰포트 최적화 (Intersection Observer 적용):**
   * 사용자가 스크롤을 내려 메인 히어로 영역이 화면에서 보이지 않게 되면, **3D 렌더링 루프를 즉시 일시 정지(Pause)**합니다. 화면 밖에서 배터리나 GPU 자원이 낭비되는 것을 100% 방지합니다.
3. **동적 해상도 스케일링 (Dynamic Pixel Ratio):**
   * 고해상도(Retina) 모바일 기기나 4K 모니터에서 풀 사이즈 3D 렌더링은 부담이 큽니다.
   * `window.devicePixelRatio`를 최대 `1.5 ~ 2`로 캡(Cap)을 씌워 제한함으로써, 사람의 눈으로 체감되는 화질 저하 없이 그래픽 연산량을 절반 이하로 줄입니다.

## 3. 요약

이 기획안은 **"가장 시각적으로 압도적이면서도, 내부적으로는 메모리 누수 0%에 도전하는 가장 가벼운 WebGL 아키텍처"**를 구축하는 것을 목표로 합니다. 승인해 주시면 이 기준에 맞춰 본격적인 코어 마이그레이션을 시작하겠습니다.
