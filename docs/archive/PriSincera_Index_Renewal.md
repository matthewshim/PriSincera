# PriSincera Index 페이지 개편 기획 (Personal Branding -> Web Service Platform)

## 1. 개편 배경 및 목표
- **현황:** 현재 PriSincera의 메인(`index`) 페이지는 개인의 커리어 철학과 이력을 중심으로 전개되는 '퍼스널 브랜딩 포트폴리오' 성격이 강합니다 (Belief → Journey → Work).
- **문제점:** Daily Digest와 Pace Note라는 실제 동작하는 핵심 웹 서비스가 런칭되었음에도 불구하고, 단순히 '포트폴리오 카드' 중 하나로 나열되어 있어 신규 서비스의 가치와 전환(구독, 로그인)을 강력하게 유도하기 어렵습니다.
- **목표:** '개인의 이력'을 축소(Journey 간소화)하고, '실제로 제공하는 가치와 서비스'(Work 강화)를 전면으로 내세워 **서비스 중심의 웹 플랫폼**으로 랜딩 페이지의 흐름을 재설계합니다.

---

## 2. 레이아웃 재구성 전략 (Section-by-Section)

### A. Hero & Belief Section (기존 유지)
- **개편 방향:** 기존의 톤앤매너와 철학 메시지가 여전히 유효하므로, 해당 섹션의 비주얼과 키 메시지는 현재 상태를 그대로 유지합니다.

### B. Journey Section (삭제 대신 '스토리 브릿지'로 간소화)
- **개편 방향:** 단순한 연도별 나열(2004~2024) 방식은 과감히 축약하고, **'이 서비스를 만들게 된 진정성(The Reason Why)'**을 보여주는 스토리텔링 영역으로 재편.
- **UI 변경 방안:**
  - 기존 7개의 타임라인을 3단계(과거의 깨달음 → 현재의 AI 전환 → 메이커로서의 시작)로 압축.
  - "20년의 IT 경험이 AI 기술을 만나, 당신의 성장을 돕는 도구로 탄생했습니다."라는 하나의 핵심 문장으로 요약.
  - 스크롤을 내리며 자연스럽게 하단의 Work(서비스 소개)로 시선이 이어지도록 시각적 브릿지(화살표 또는 빛줄기 등) 추가.

### C. Work Section (메인 서비스 쇼케이스로 승격 및 강화 ⭐️)
- **개편 방향:** 현재의 3단 '프로젝트 카드 그리드'를 해체하고, 각 서비스(Daily Digest, Pace Note)가 하나의 독립된 **플래그십(Flagship) 섹션**으로 돋보이도록 공간을 대폭 할당합니다.
- **상세 구성 방안:**
  1. **Daily Digest (Curation & Learning)**
     - 좌/우 분할 레이아웃 적용 (좌측 텍스트, 우측 앱 목업/대시보드 UI).
     - 메인 페이지에서 즉시 구독할 수 있는 CTA 버튼(`✨ 하루 5분, 다이제스트 무료 구독`)을 전면에 배치.
  2. **Pace Note (Action & Branding)**
     - 조급함을 덜어주는 '마이크로 트래커'의 시각적 요소를 강조 (주차별 캘린더 타임라인 UI 등).
     - 행동을 유도하는 CTA 버튼(`✨ 3초 만에 로그인하고 나만의 궤도 만들기`) 배치.
  3. **PriSincera Base (Foundation)**
     - '이 사이트 자체가 브랜딩 결과물이다'라는 기존 PriSincera 카드는 각 서비스 소개가 끝난 하단에 서브 카드로 작게 배치하여 톤 다운.

### D. Connect Section (통합 Call to Action)
- 단순 연락처/이메일을 남기는 공간에서 나아가, "PriSincera와 함께 매일 조금씩 성장해 보세요"라는 통합 온보딩 영역으로 진화시킵니다.

---

## 3. 기대 효과
1. **유저 여정(UX) 최적화:** 개인의 이력서(Resume)를 읽는 느낌에서, 유용하고 가치 있는 서비스(SaaS)를 둘러보고 즉시 구독/로그인하는 능동적 경험으로 탈바꿈합니다.
2. **트래픽 전환율(CVR) 상승:** 메인 페이지에서 각 서비스의 핵심 기능과 CTA 버튼이 직접 노출되므로, 서브 페이지로 이동해야만 했던 기존 대비 이탈률이 감소하고 가입 전환이 늘어납니다.
3. **브랜딩 진화:** 개인의 과거(Journey)보다 개인이 만들어낸 결과물(Work)이 주인공이 되며, '증명하는 메이커'로서의 전문성이 한층 강화됩니다.

2. **Phase 2:** `WorkSection.jsx`를 개편하여 Daily Digest와 Pace Note를 각각 별도의 풀-와이드(Full-wide) 레이아웃으로 디자인.
3. **Phase 3:** 각 서비스 섹션에 CTA 버튼을 배치하고 랜딩 페이지에서의 인터랙션(스크롤 등장 효과 등) 최적화.
