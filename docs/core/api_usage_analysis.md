## 1. 개요 (Background)

PriSincera 플랫폼은 기술 블로그인 **Builder's Log** 작성 시 본문 내용과 밀접하게 연관된 GitHub 커밋 내역을 AI가 자동 추출하도록 설계된 기능부터, 매일 아침 전 세계 비즈니스/테크 동향을 요약·합성하여 송신하는 **Daily Digest** 파이프라인에 이르기까지 **Gemini API**를 다방면에서 핵심 지능으로 가용하고 있습니다.

기존 파이프라인의 안전 가용량(Rate Limit) 수준은 넉넉하게 검증되었으나, 실제 라이브 운영에 들어선 후 매일 백그라운드 크론 잡이 가동되는 과정에서 **결제 계정이 연동된 Paid Tier의 단가 및 무방비한 Fallback 시스템**으로 인해 요금 누출이 발생하는 지점이 포착되었습니다. 

본 문서는 서비스의 완성도를 훼손하지 않으면서도 API 사용 효율을 극대화하여 **100% 무료 제공량(Free Tier) 범위 내에서 비용 부담 없이 무한 가동할 수 있는 4대 아키텍처 개선 방안 및 요금 분석 리포트**를 수록합니다.

## 2. API 의존성 및 가격 구조 정밀 해부

### A. 모델 체급별 과금 정책 비교 (구글 AI Studio 기준)
- **`Gemini 1.5 Flash`**: 극도의 고속 처리가 가능하며, API 무료 단가가 파격적이고 유료 단가 역시 초저가입니다.
  - **입력 단가:** \$0.075 / 1,000,000 토큰
  - **출력 단가:** \$0.30 / 1,000,000 토큰
- **`Gemini 1.5 Pro`**: 복잡한 추론에 최적화되었으나, Flash 체급 대비 **약 16.6배 비싼 단가**를 가집니다.
  - **입력 단가:** \$1.25 / 1,000,000 토큰
  - **출력 단가:** \$5.00 / 1,000,000 토큰

> [!WARNING]
> **결제 계정 연동(Paid Tier - 후불)의 함정**
> 구글 AI Studio 프로젝트에 결제 계정을 연결하여 유료 등급(`Tier 1`)으로 활성화되면, 해당 프로젝트의 API Key를 활용하는 모든 호출은 무료 티어 제공량을 초과하지 않았더라도 **단 1토큰의 입력부터 유료 요율로 즉시 과금이 개시**됩니다.
> 따라서, 파이프라인의 불필요한 호출 빈도를 억제하고 단가가 높은 Pro 모델로의 백업 시도를 물리적으로 제거하는 최적화 엔지니어링이 선행되어야 비용을 ₩0원 수준으로 수렴시킬 수 있습니다.

## 3. 누수 원인 진단 및 최적화 처방 (4대 개선 사항)

```
                 [ API 최적화 다이어트 아키텍처 ]
                 
  +------------------+     +------------------+     +-------------------+
  | 1. Pro 모델 배제 | ==> | 2. 단일 모델 순차| ==> | 3. 배치 3배 확대 |
  | (유료 요인 차단) |     |  (호출 중복 격감)|     |  (5개 -> 12개)    |
  +------------------+     +------------------+     +-------------------+
                                                              │
                                                              ▼
                                                    +-------------------+
                                                    | 4. RPM Throttle  |
                                                    | (무료 15RPM 통과) |
                                                    +-------------------+
```

### 1) 비싼 Pro 모델 (`Gemini Pro`) 완전 배제
- **진단:** 네트워크 장애나 Rate Limit으로 인해 `gemini-2.0-flash` 호출이 지연되거나 실패하면, 모델 목록의 예비 모델인 `gemini-1.5-pro-latest`가 자동 호출되어 건당 수백 원 이상의 높은 요율로 즉시 과금되었습니다.
- **조치:** 텍스트 분류, 아티클 요약, 다국어 완역 수준의 태스크는 최신 Flash 모델로도 100% 동일한 품질을 낼 수 있으므로, 모델 후보군 리스트에서 Pro 계열 모델을 완전 영구 삭제하였습니다.
- **적용 대상:** `pipeline/src/lib/gemini.mjs`, `admin-api.mjs`

### 2) 재시도 알고리즘 경량화 (Multi-Model Spraying 수정)
- **진단:** 단 1회의 전송 실패 발생 시에도, 3회의 재시도 기회마다 5개의 모델군을 무차별 연속 난사(`3 * 5 = 총 15회 호출`)하여 불필요한 트래픽 낭비를 가져왔습니다.
- **조치:** 1회 시도마다 오직 **1개의 최적 Flash 단일 모델만 순차 매핑하여 대입**하고, 실패 시 쿨타임(1.5초) 후 차선책 모델로 넘어가도록 개선하여 오작동 트래픽을 80% 이상 절감시켰습니다.

### 3) 아티클 스코어링 배치(Batch) 크기 대폭 확대 (5 ➡️ 12)
- **진단:** 수집된 아티클 후보군을 5개씩 나누어 API를 날려와, 매일 아침 데일리 합성(Daily Run) 시 고정적으로 수십 회의 중복 호출이 발생했습니다.
- **조치:** Flash 모델의 거대한 컨텍스트 윈도우 한도를 유기적으로 활용하여 배치 크기를 **12개**로 3배 가까이 확대했습니다.
- **효과:** 일일 API 호출 횟수가 **절반 이하**로 줄어들었으며, 반복 전달되던 시스템 프롬프트(System Prompt Overhead)의 소모량도 크게 아꼈습니다.

### 4) 무료 RPM 방어를 위한 정밀 시간차(Throttle) 확보 (15 RPM 우회)
- **진단:** 결제 연동을 정지하고 100% 무료 티어로 되돌아갔을 시, 구글은 분당 15회 요청(15 RPM) 한도를 강제 적용합니다. 루프 처리가 빠르게 돌면 쿼터 초과 에러(`429`)로 이메일 발송이 굳을 수 있습니다.
- **조치:** 배치 루프 간 대기 시간을 기존 1.5초에서 **4초(4000ms)**로 안전하게 확장 조정하여 무료 트래픽 속도 제한을 유유히 통과하도록 성형했습니다.

## 4. 종합 요금 시뮬레이션 및 안전성 평가

개선된 Flash 단일 전용 파이프라인의 실질 일일 사용 토큰을 기반으로 과금 시뮬레이션을 돌려 비용 타당성을 검증했습니다. (1일 아티클 수집 후보량 60개 가정)

### 📈 일일 파이프라인 토큰 점유율 추정치
- **아티클 스코어링 (배치 12개 적용):**
  - 입력 토큰: 5회 호출 × (프롬프트 + 12개 요약본) ≈ **약 55,000 토큰**
  - 출력 토큰: 5회 호출 × (12개 점수 JSON) ≈ **약 8,000 토큰**
- **에디터 코멘트 및 노트 요약:**
  - 입력 토큰: 선정 아티클 원문 분석 ≈ **약 7,000 토큰**
  - 출력 토큰: 요약 및 노트 완성 ≈ **약 1,500 토큰**

### 💳 유료 결제 유지 시 청구 비용 시뮬레이션
- **일일 입력 토큰 요금:** 62,000 토큰 × \$0.000000075 = **\$0.00465** (약 6원)
- **일일 출력 토큰 요금:** 9,500 토큰 × \$0.00000030 = **\$0.00285** (약 4원)
- **일일 총비용:** **\$0.0075 (약 10원)**
- **월간(30일) 예상 총 과금액:** **\$0.225 (약 300원 미만)**

### 🏁 최종 가용성 권고사항
- **100% 무료 등급 이관 반영 완료**: v1.1 설계 당시에는 월 300원 미만의 소액 청구로 유료 결제 유지를 권장했으나, 기업 환경의 요금 관리 및 결제 리스크 완전 원천 차단을 위해 **결제 연동이 없는 100% 독립 무료 등급(Free Tier) API Key로 이관**을 진행하였습니다.
- 한도 초과 시에도 요금이 결제되지 않고 단순히 API 에러가 반환되는 무료 등급의 특성상 비용 청구 리스크가 0%로 완벽히 방어됩니다.

## 5. v1.2 업데이트: 무료 등급(Free Tier) 완전 이관 (2026-06-08)

### 1) 구글 Workspace 조직 계정 무료 쿼터 제한 대응
- **진단**: Google Workspace 조직 도메인 계정(`@prisincera.com`)으로 생성한 구글 클라우드 프로젝트는 결제 수단이 비활성화된 경우 보안 및 남용 방지 정책상 무료 쿼터가 `0`으로 제한되는 현상이 발견되었습니다.
- **조치**: 조직 도메인 대신 결제 수단이 등록되지 않은 **순수 개인 구글 계정(`@gmail.com`)**을 활용하여 Google AI Studio에서 전용 무료 API 키를 새로 발급받아 연동을 처리하였습니다.

### 2) API v1beta 모델명 버전 호환성 패치
- **진단**: 무료 API 키 적용 과정에서 `gemini-1.5-flash` 모델은 `v1beta` 엔드포인트 상에서 `404 Not Found` 에러를 반환하고, `gemini-2.0-flash` 모델은 신규 개인 계정 무료 한도에서 제한(limit: 0)되는 현상이 확인되었습니다.
- **조치**: 무료 등급에서 완벽하게 동작하며 안정적인 응답을 보장하는 `gemini-flash-latest`와 `gemini-2.5-flash` 모델로 파이프라인(`gemini.mjs`) 및 어드민 백엔드(`admin-api.mjs`)의 모델 우선순위 리스트를 전격 갱신하였습니다.

### 3) API Key 노출 방지 및 보안 무결성 확보
- **보안 수칙 준수**: 연동된 API Key는 소스 코드, 클라이언트 환경(.env) 또는 Git 커밋 이력에 절대로 일반 텍스트 형태로 노출되거나 저장되지 않도록 철저히 차단하였습니다.
- **Secret Manager 통합**: 새로 발급받은 실무 API Key는 오직 GCP Secret Manager의 `GEMINI_API_KEY` 및 `GEMINI_ADMIN_API_KEY` 보안 비밀의 최신 버전(`latest`)으로만 업로드되어 있으며, Cloud Build 배포 및 Cloud Run 구동 런타임에 동적으로 주입됩니다.
