## 1. 사업 배경 및 핵심 가치 제안 (Value Proposition)

### 1.1. 배움(Input) - 실행(Action) - 복기(Feedback)의 완벽한 순환
성장하는 전문가는 단순히 공부하거나 할 일을 처리하는 데 시간을 쓰는 것을 넘어, **정보를 소비(Daily Digest)하고, 이를 바탕으로 주간의 핵심 의사결정을 트래킹(Pace Note)하며, 이를 정기적으로 복기(Captain's Log)하는 완벽한 성장 플라이휠(Growth Flywheel)**이 필요합니다.
Pace Note는 단순한 개별 도구의 모음이 아닌, **"매일의 배움(Input)이 주간의 행동(Action)이 되고, 한 주간의 실행(Action)이 깊은 성찰(Feedback)이 되며, 그 성찰(Feedback)이 다시 다음 주의 개인화된 배움(Input)을 정교하게 튜닝하는 데이터 기반의 상호 연결 순환 구조"**를 통해 "제로부터 다시 배우는 러너 (Learner & Runner)"들의 압도적인 커리어 성장을 가이드합니다.

### 1.2. 왜 macOS 네이티브 데스크톱 앱인가? (비용 0원 & 보안 극대화)
* **데스크톱 상주를 통한 습관 형성**: 브라우저 탭에 파묻히지 않고, 메뉴바 상주와 시스템 단축키(`⌥+Space` 또는 `⌘+K`) 호출을 통해 실무와 생각 정리를 오가는 매끄러운(Frictionless) 기록 경험을 선공개합니다.
* **완벽한 로컬-퍼스트 (서버리스) 아키텍처**: 모든 주간 궤도(오빗)와 회고 데이터(성찰 일지)는 **서버가 아닌 사용자의 맥북 로컬 파일시스템에만 암호화 저장**됩니다. 
* **개발사 서버 유지비 0원**: 중앙 서버나 데이터베이스를 두지 않으므로 데이터 보관 비용 및 해킹/유출 리스크가 원천 차단됩니다. 기기 간 동기화는 애플의 기본 인프라인 **iCloud Drive**를 통해 무료로 자동 처리됩니다.

## 2. 정교한 타깃 고객 정의 (Target Audience Analysis)

Pace Note는 단순한 인구통계학적 구분을 넘어, **IT/테크 씬에서 '역량을 초고속으로 흡수·축적해야 하는 시기(주니어-미드)'와 '과거의 성공 방식을 버리고(Unlearn) 의사결정을 재정의해야 하는 시기(시니어-리더)'의 상호 연결성**을 타깃으로 정의합니다. 특히 AI의 도래로 매일 새로운 기술을 제로부터 다시 배워야 하는 **제로스타트(Zero-Start) 마인드셋**을 가진 지식 근로자들을 깊게 공략합니다.

```
                      [ IT 커리어 성장 주기의 상호 연결성 ]
주니어-미드 트랙 (역량 축적기) ───────────────────> 시니어-리더 트랙 (역량 재정의기)
- 실무 구현 기술의 깊이 지향                         - 아키텍처 및 의사결정의 리스크 관리
- AI 도구를 활용한 초고속 생산성 확보                  - 레거시 unlearning 및 리더십 피드백
                    │                                          │
                    └─────────────────── 공통 분모 ────────────────────────┘
                       "제로부터 다시 배우는 러너 (Zero-Start Learner)"
```

### 2.1. 세그먼트 A: 주니어-미드 트랙 (Fast Learner - 2030 주도적 테크 성장러)
* **정의**: 실무 기술 스택(프론트엔드, 백엔드, 데이터, AI 엔지니어링)을 빠르게 축적하며 스스로의 아키텍처 구현 역량을 증명해야 하는 개발자, 디자이너, 주니어 기획자.
* **Pace Note의 순환 시나리오**: 
  * **[Input (배움)]**: 오늘 아침 배포된 `React 19 Server Actions` 기술 시그널을 읽고, 하단의 실전 액션 챌린지 버튼을 눌러 주간 오빗에 등록합니다.
  * **[Action (실행)]**: 오프라인 SQLite 환경의 단축키 HUD 창을 통해 등록된 Server Actions 실무 구현 단계를 매일 체크하며 실행합니다.
  * **[Feedback (복기)]**: 주말에 회고를 열면 이번 주 완료한 Server Actions 내역이 자동 로드됩니다. AI 코치는 사용자가 작성한 일지를 파악해 로컬 키체인 API를 통해 원가 비용으로 정밀한 코칭을 제공하고, 로컬 DB에 "예외 처리 및 성능 튜닝" 보완 필요 태그를 자동 주입합니다.
  * **[Next Input (개인화)]**: 다음 주 월요일, 로컬 가중치 알고리즘에 의해 `React/Next.js Error Boundary & Handling` 고급 테크 시그널 카드가 유저 피드의 최상단에 우선 배치됩니다.

### 2.2. 세그먼트 B: 시니어-리더 트랙 (Re-inventor - 40대 이상 제로스타터 리더)
* **정의**: 팀의 아키텍처를 결정하고 기술 부채와 프로덕트 우선순위를 관리하지만, AI 시대의 대전환으로 인해 과거의 개발 패러다임을 지우고 스스로 '제로부터 다시 배우겠다'는 의지가 확고한 VP, CTO, 테크 리드, 1인 메이커.
* **Pace Note의 순환 시나리오**:
  * **[Input (배움)]**: `글로벌 테크 기업의 기술 부채 청산 케이스 스터디` 시그널 및 의사결정 프롬프트를 소비하고, 단 한 번의 클릭으로 이번 주 오빗(의사결정 보완 액션)에 등록합니다.
  * **[Action (실행)]**: 팀의 아키텍처 로드맵 중 리스크가 높은 레거시 모듈을 측정하는 1차 분석 작업을 완료하고 오빗 트래커에 체크합니다.
  * **[Feedback (복기)]**: 주간 리더십 회고 패널에 해당 의사결정 로그가 자동 임포트됩니다. 사용자는 아키텍처 판단에 대한 AI 코칭을 받으며 장기 히스토리를 요약해 Growth Portfolio로 추출합니다. AI 코칭 단에서 "의사결정의 리스크 마이그레이션 및 커뮤니케이션 분쟁 해결"에 대한 병목이 인지되어 로컬 데이터에 키워드가 저장됩니다.
  * **[Next Input (개인화)]**: 다음 주 월요일, `엔지니어링 리드 커뮤니케이션 및 아키텍처 리스크 제어 기법`에 관한 글로벌 시그널이 우선 노출됩니다.

### 2.3. 공통 분모 (Core Commonality)
두 세그먼트는 모두 **macOS 기반의 프리미엄 지식 노동 생태계**에 속해 있으며, "정리되지 않은 정보는 노이즈에 불과하고, 복기하지 않는 지식은 도태된다"는 깊은 인식을 공유합니다. 이들은 매 순간 제로부터 학습하며(Learner) 달리는(Runner) **'러너'**들입니다.

## 3. 어플리케이션의 핵심 방향성 및 기능 정의 (Core Features)

### 3.1. 화면 구성 및 UX 콘셉트
* **메뉴바 상주형 미니 팝업 (Mini-Dashboard)**: 메뉴바 아이콘 클릭 시 슬라이드 다운되는 컴팩트 뷰. 매일 아침의 Daily Digest 키워드와 오늘 완료해야 할 오빗(Orbit)을 미니멀하게 제공.
* **HUD(Heads-Up Display) 스타일의 메인 윈도우**: 단축키 호출 시 화면 중앙에 반투명 글래스모피즘(Glassmorphism) 효과가 적용된 미려한 메인 화면 활성화. 

### 3.2. 핵심 제공 기능 (서버리스 & 로컬 퍼스트 구현)
1. **Daily Personalized Tech Signal (트랙별 맞춤형 고밀도 IT 테크 피드)**:
   * **3초 로컬 온보딩 개인화 필터**: 회원가입이 없는 로컬 퍼스트 원칙을 유지하며, 앱 최초 실행 시 사용자가 **[수준별 트랙 (Junior-Mid / Senior-Leader)]**과 **[관심 테크 영역 (AI/LLM, System Design, DevOps, Tech Lead/PM)]**을 선택할 수 있는 설정 모달 UI를 제공하며, 정보는 로컬 SQLite에 안전하게 격리 저장됩니다.
   * **트랙별 콘텐츠 다변화**:
     * **IT 테크 시그널 (IT Tech Signal)** 및 **실무 AI 프롬프트 (AI Workstation)**를 눈높이에 맞춰 **Junior-Mid(실무 구현, 디버깅, 생산성)**와 **Senior-Leader(아키텍처 설계, 기술 부채, 리더십/팀 관리)**로 이원화 공급합니다.
   * **정적 CDN 및 온디바이스 로컬 랭킹 가중치 필터링**:
     * 매일 새벽 배치 파이프라인(`tech-composer.mjs`)을 통해 생성된 GCS 내 정적 JSON 피드(`daily/junior_${date}.json`, `daily/senior_${date}.json`)를 클라이언트가 다운로드한 뒤, **로컬 SQLite에 저장된 지난주 회고(Captain's Log)의 AI 피드백 취약점 키워드 가중치를 대조**하여, 사용자가 오늘 읽어야 하는 가장 시급한 기술 요약 카드를 피드 맨 앞자리에 정적으로 큐레이션합니다.
2. **Interactive Orbit Tracker (Pace Note & "Click-to-Orbit" 연계)**:
   * **원클릭 액션 변환 ("Click-to-Orbit")**: 데일리 시그널 카드 하단에 위치한 **[오빗 추가 (Add to Orbit)]** 버튼을 누르면, 해당 테크 정보의 실전 적용을 위한 구체적인 **실무 액션 챌린지 템플릿 (3개의 세부 할 일 Task가 프리필된 구조)**이 사용자의 오빗 트래커에 가입 절차 없이 즉각 주입됩니다.
   * 로컬 SQLite 데이터베이스를 기반으로 동작하여 인터넷 연결이 완전히 차단된 비행기나 지하철 안에서도 오빗 관리 및 태스크 체크박스 조작이 가능합니다.
3. **Smart Captain's Log (주간 회고 및 복기 - "Action-to-Feedback" 자동 컨텍스트 연계)**:
   * **"Action-to-Feedback" 자동 컨텍스트 연계**: 매주 일요일 회고 패널을 활성화하면, 사용자가 이번 주에 달성한 오빗 목록과 달성하지 못한 오빗 목록, 그리고 챌린지 분야의 키워드 정보가 **성찰 패널에 정형 데이터 컨텍스트로 자동 주입**됩니다. 사용자는 "이번 주에 내가 무엇을 했지?" 고민하며 백지에서 생각할 필요 없이, 이미 시스템이 바인딩해둔 활동 데이터를 기반으로 즉각적이고 날카로운 자기성찰을 기록하게 됩니다.
   * **PRO 전용 "나만의 종신 AI 코치" (BYOK 모델)**: 암호화 저장된 사용자 개인의 API Key를 이용해 로컬 기기 내부에서 인공지능 엔드포인트와 다이렉트 통신하여 프라이버시가 100% 보장된 의사결정/성장 피드백을 산출합니다.
   * AI 피드백 쿼리 완료 시, LLM 분석 단독 결과물 내에 포함된 핵심 보완 영역 키워드(예: `Clean Code`, `Latency Optimization`, `Tech Debt Measurement`)를 분리 파싱하여 로컬 SQLite의 **가중치 우선순위 테이블(Keyword Priority Table)**에 갱신/기록함으로써 다음 주 데일리 피드 큐레이션과 유기적으로 바인딩합니다.
4. **AI Growth Portfolio Builder**:
   * 축적된 주간 데이터와 사색 기록을 기반으로 한 고품질 PDF 및 마크다운(.md) 파일 로컬 생성/변환 기능 지원 (로컬 라이브러리 활용으로 백엔드 렌더링 서버 미필요).
5. **Raycast / Alfred 확장 연동**:
   * 로컬 CLI 및 URL Scheme을 열어두어 사용자가 Raycast나 Alfred로 간편하게 오늘 할 일을 추가/복기할 수 있도록 연동.

## 4. 유료화 전략 (Monetization Strategy)

개발사의 고정 유지비(서버비, AI API 호출비)가 발생하지 않기 때문에, 구독형 강제가 아닌 **'일회성 영구 구매(One-time Purchase)'** 라이선스 모델을 중심으로 설계하여 앱 유료 결제 전환 장벽을 대폭 낮춥니다.

```
[무료 버전 (Free Basic)]
- 당일 제공되는 정적 Daily Digest 콘텐츠 조회
- 로컬 주간 궤도 관리 (주당 3개 태스크 제한)
```

### 4.1. PRO 라이선스 API Key 연동(BYOK)의 핵심 유저 메리트 (User-Centric Value Proposition)

1. **구독 피로도(SaaS Fatigue)의 완전한 해방과 평생 소유**:
   * 매달 $15~$20씩 통장에서 빠져나가는 넷플릭스, 노션, ChatGPT 등의 구독 요금제에 지친 유저에게 **"단 한 번 결제($5.99)로 평생 소장"**하는 평화를 제공합니다.

2. **서비스 종료(Bankrupt) 리스크 제로 — 내 인생 기록의 영구적 보존**:
   * 일반 클라우드 기반 SaaS는 회사가 망하면 서버가 내려가고 사용자가 몇 년간 쌓아온 기록과 포트폴리오가 한순간에 증발합니다.

### 4.2. 설치 유저의 PRO 결제 전환율 극대화 전략 (Funnel & Conversion Optimization)
어플리케이션은 웹 페이지 방문에 비해 **'다운로드 -> 설치 -> 권한 승인 -> 실행'**까지의 허들(Funnel Drop-off)이 매우 높은 것이 사실입니다. 따라서, **어려운 허들을 뚫고 앱을 실행한 소중한 유저가 이탈 없이 즉시 $5.99 PRO 결제로 전환하도록 설계한 인앱 전환 장치**들입니다.

1. **실행 30초 내 '아하 모먼트(Aha! Moment)' 선사 (가입 없는 즉시 실행)**:
   * 앱을 켜자마자 회원가입이나 이메일 인증 절차 없이, **오늘 아침 배포된 글로벌 IT 뉴스(Daily Signal)와 실무 프롬프트가 글래스모피즘 HUD 화면에 즉시 로드**됩니다. 유저는 설치 직후 단 30초 만에 이 앱이 나에게 주는 정보적 가치를 시각적으로 인지하게 됩니다.
2. **소프트 게이트(Soft-Gate)를 통한 '행동 소유권(Endowment Effect)' 자극**:
   * **주당 3개 궤도(할 일) 무료 제한**: 사용자는 1~3개의 핵심 목표를 자유롭게 등록해 트래킹하며 앱의 유용성을 체감합니다. 4번째 목표(예: 독서, 영어 공부, 운동에 이어 '업무 루틴' 추가 시)를 등록하려는 순간, 자연스럽게 결제 안내가 나타납니다. 유저는 이미 공들여 설정한 3개의 목표를 계속 유지하고 완성하고 싶은 심리(소유 효과)로 인해 $5.99의 가벼운 결제를 진행하게 됩니다.
3. **AI 코칭 '티저(Teaser)' 및 블러(Blur) 효과**:
   * 무료 유저가 주간 성찰 노트를 작성하면, AI 피드백 화면에 일부 키워드를 제외한 피드백 영역이 반투명 블러(Blur) 처리되어 노출됩니다: *"당신의 종신 AI 코치가 성장의 병목 현상과 심리 상태 분석을 완료했습니다. 코칭을 해제하고 이번 주 성장의 액션 아이템을 확인하세요."*
   * 1회 무료 체험권 등을 통해 단 한 번 AI 코칭의 날카로운 품질을 맛보게 한 후, 다음 주부터는 결제 시에만 열리도록 유도하여 가치 중심의 결제를 유도합니다.
4. **Touch ID 기반의 원클릭 네이티브 결제 (Frictionless Payment)**:

   - **API Key 가이드 화면 (`SylphioApiKeyGuide.jsx`, `SylphioApiKeyGuide.css`)**: 사용자가 개인 API Key를 쉽게 획득할 수 있도록 안내하는 기존 가이드 컴포넌트를 레이아웃 수정 없이 그대로 재활용합니다.

   - **일일 다이제스트 컴포저 (`composer.mjs`)**: 새벽 주기적 배치 스케줄러를 기반으로 테크 뉴스를 수집하고, 실무 프롬프트를 생성해 내는 일일 콘텐츠 파이프라인을 그대로 유지하여 Cloud 인프라에서 구동합니다. 결과물 정적 JSON 파일을 GCS CDN 버킷에 동기화하여 클라이언트가 direct HTTP GET 요청으로 트래픽 비용 $0에 다운로드하도록 연동합니다.

### 6.2. 웹서비스의 조정 및 리팩토링 범위 (Modification Scope of Existing Web Assets)
로컬-퍼스트 환경의 데스크톱 앱과 수준별 테크 트랙 전환, 그리고 성장 플라이휠의 액션 챌린지 자동 생성을 지원하기 위한 기존 웹서비스 측의 변경 범위입니다.

1. **데이터 계층 디펜던시 추상화 (React Custom Hooks & Providers)**:
   - **[현황 정정] 현재 구조 = REST API 경유 (Firestore 직접 의존 아님)**: 기존 웹 프론트엔드(`PaceNoteDashboard.jsx`)는 Firebase SDK로 Firestore를 직접 읽고 쓰지 않습니다. 실제로는 인증 토큰(`Authorization: Bearer <idToken>`)을 실어 백엔드 REST API(`/api/pacenote/*`, `server.mjs`)를 호출하며, Firestore는 서버 측(`server.mjs`, `admin-api.mjs`, `pacenote-composer.mjs`)에서만 사용됩니다. 따라서 "클라이언트 Firebase SDK 차단" 작업 부담은 사실상 없고, 데이터 호출부가 이미 REST 인터페이스로 분리되어 있어 추상화의 절반은 완료된 상태입니다.
   - **[핵심 작업] 무인증 로컬 단일 사용자 데이터 모델 변환**: 진짜 난점은 SDK 분리가 아니라 **인증 모델의 제거**입니다. 현재 웹 PaceNote는 Firebase Auth 로그인이 필수이며, 데이터는 `pacenotes/{userId}/weeks/...` 형태의 **유저별 Firestore 도큐먼트 모델**입니다. 데스크톱은 **무인증·로컬 단일 사용자** 원칙을 채택하므로, (a) `idToken`/로그인 분기를 전면 제거하고, (b) 유저별 컬렉션 구조를 단일 사용자 기준의 로컬 SQLite 테이블(`orbits`, `tasks`, `retrospectives`, `keyword_weights`)로 매핑하는 **데이터 형상 변환 어댑터**를 신규 설계합니다.
   - **`usePaceNoteData` 커스텀 훅 설계**: 프론트엔드가 구동 환경을 자동 감지하여, 웹 브라우저일 때는 기존의 인증 기반 REST API(`/api/pacenote/*`)를 호출하고, Tauri 데스크톱 환경일 때는 토큰/로그인 절차 없이 Tauri IPC 브릿지(`window.__TAURI__.invoke`)로 로컬 SQLite 데이터를 처리하도록 추상화 인터페이스를 일원화합니다. 두 경로가 동일한 데이터 형상(Data Shape)을 반환하도록 클라이언트 측 정규화 매퍼를 둡니다.
2. **수준별 IT 콘텐츠 파이프라인 추가 및 스키마 확장**:
   - **[결정 반영] `study-composer.mjs` 존치 + `tech-composer.mjs` 신규 추가**: 기존 어학 전용 파이프라인(`study-composer.mjs`)은 **웹 전용 기능으로 그대로 유지(삭제하지 않음)**합니다. 별도로, 주니어급(실무 구현) 및 시니어급(아키텍처 의사결정) 수준별 테크 시그널과 AI 프롬프트를 가공해 내는 **`tech-composer.mjs`를 신규로 추가**합니다. 데스크톱 앱은 이 테크 파이프라인 산출물만 소비합니다.
   - **액션 챌린지(Action Challenge) 스키마 빌딩**: `tech-composer` 배치 작업 시 각 데일리 카드 하단에 연동될 **원클릭 액션 챌린지 스키마 (`action_challenge` 객체 - 3개 세부 할 일 Task 포함)**를 Gemini Flash AI가 자동으로 분석 및 구조화하여 JSON 내에 삽입합니다.
   - **정적 JSON CDN 스키마 가산 확장**: 기존 `daily/${date}.json`(`signal` + `study`)을 보존한 채, 수준별 트랙용 `daily/junior_${date}.json` 및 `daily/senior_${date}.json` 과 관심 도메인 필터링용 메타 데이터 스키마를 **병렬로 추가 빌딩**합니다. (웹 어학은 기존 파일을, 데스크톱은 트랙 파일을 읽음)
3. **어드민 콘텐츠 실시간 GCS 동기화 연동 (`admin-api.mjs`)**:
   - **[현황 정정] `syncDailyGcs`는 신규 개발 (기존 함수 아님)**: 현재 `admin-api.mjs`는 GCS에서 데이터를 **읽기(read)만** 하며, 어드민 수정 내용을 GCS로 쓰는 `syncDailyGcs` 헬퍼는 존재하지 않습니다. GCS 쓰기(write) 로직은 새벽 배치 파이프라인(`composer.mjs` Phase 5)에만 있고, 단일 `daily/${date}.json`을 배포하는 형태입니다.
   - **`syncDailyGcs(dateStr)` 신규 구현**: 관리자가 어드민 패널에서 콘텐츠 및 액션 챌린지 할 일 항목을 수정하면, `syncDailyGcs`가 작동하여 GCS 내 해당 날짜의 2개 트랙별 JSON 파일(`junior_${date}.json`, `senior_${date}.json`) 및 통합 인덱스(`index.json`)를 즉시 업데이트하고 Cloudflare 캐시를 퍼지하도록 `admin-api.mjs`에 신규 헬퍼를 추가합니다. (composer.mjs Phase 5의 GCS write 유틸을 공통 모듈로 추출해 재사용)

### 6.3. 데스크톱 앱 출시를 위한 추가 신규 개발 범위 (New Development Scope for macOS App)
macOS 데스크톱 환경의 시스템 기능(메뉴바 상주, 단축키 HUD, iCloud 동기화 등)과 영구 결제(StoreKit IAP), 그리고 로컬 AI 성찰 및 가중치 기반 순환 랭킹 엔진을 실현하기 위한 신규 개발 태스크입니다.

1. **Tauri 프로젝트 및 macOS 전용 Vibrancy / HUD 윈도우 셋업**:
   - Tauri v2 환경에서 `tauri.conf.json` 내의 윈도우 빌더 설정을 구성하고, macOS 네이티브 Vibrancy 효과(흐림 효과 및 투명도)를 적용하기 위해 Cocoa 윈도우 속성(`NSVisualEffectView` 및 `NSWindowVibrancy`) 활성화.
   - 글로벌 숏컷(`⌥+Space`)을 OS 전역 키 리스너에 등록하고, 메인 HUD 창을 토글 표시. `window.on_blur` 이벤트를 리스닝하여 사용자가 창 밖을 클릭하면 자동으로 페이드아웃 및 윈도우 숨김(`window.hide()`)을 보장하는 이벤트 핸들러 Rust 구현.
2. **SQLite 로컬 데이터베이스 통합 및 Tauri IPC 브릿지 설계**:
   - Rust 백엔드에 `sqlx-sqlite` 또는 `rusqlite` 드라이버를 이식하고, 최초 앱 런타임 구동 시 사용자 로컬 스토리지에 데이터베이스 파일(`pacenote.db`)을 자동 마이그레이션(`orbits`, `tasks`, `retrospectives`, `keyword_weights` 테이블 DDL 실행).
   - Tauri IPC 명령(`#[tauri::command]`)을 통해 React 단에 비동기 CRUD 인터페이스 제공 (`get_pacenote`, `add_custom_task`, `toggle_task`, `accept_recommendation`, `save_diary`).
3. **iCloud Ubiquity Container 기반 Zero-Knowledge 자동 동기화 & 실시간 핫 리로드**:
   - SQLite DB 및 Markdown 파일들의 저장 경로를 macOS 공식 iCloud 컨테이너 경로(`~/Library/Mobile Documents/iCloud~com~prisincera~pacenote/Documents`)로 강제 매핑.
   - Rust 백엔드에서 `notify` 크레이트를 통해 이 폴더를 Recursive하게 감시하는 파일 시스템 감시 스레드를 백그라운드 구동. 다중 기기(예: 맥북과 아이맥)에서 동기화가 완료되어 SQLite DB가 업데이트되는 즉시 프론트엔드 React 측으로 `tauri://db-updated` 이벤트를 발행하여 화면을 핫 리로드. SQLite의 WAL(Write-Ahead Logging) 저널 모드를 활성화하여 동기화 중 파일 잠금(Locking) 충돌을 미연에 방지.
4. **Keychain API 연동 및 온디바이스 BYOK AI 클라이언트 개발**:
   - 사용자가 설정에서 직접 입력한 OpenAI 또는 Gemini API Key를 macOS 고유의 하드웨어 보안 스토리지인 **Keychain(키체인)**에 암호화하여 저장할 수 있도록 Rust `security-framework` 크레이트 탑재.
   - AI 피드백 쿼리 시, 사용자 키체인에서 안전하게 복호화한 API Key와 최근 6개월 치의 로컬 궤도/성찰 텍스트들을 엮어 프롬프트를 구성한 뒤, 중간 프록시를 완전히 배제하고 OpenAI/Gemini 다이렉트 HTTPS 통신을 수립하여 100% 프라이버시가 확보된 로컬 AI 코칭 결과 생성.
5. **성장 플라이휠(Growth Flywheel) 제어를 위한 신규 Rust Command 구현**:
   - **`add_orbit_from_signal(signal_data)`**: 데일리 시그널 카드 정보 및 구조화된 액션 챌린지(`action_challenge`)를 IPC로 수령하여, 로컬 SQLite 트랜잭션을 실행해 주간 오빗과 3개의 세부 Task를 일괄 주입하는 Rust 기능 구현.
   - **`compile_weekly_context()`**: 현재 주차의 SQLite 오빗 테이블에서 완료(`Checked`) 및 미완료(`Unchecked`) 항목 목록, 카테고리 태그 등을 읽어와 AI 프롬프트에 동적 삽입할 컨텍스트 문자열을 조합해 프론트엔드 및 AI 쿼리에 공급하는 기능 구현.
   - **`update_keyword_weights(bottleneck_keywords)`**: AI 코치가 도출한 유저의 기술적 취약점 키워드 리스트를 IPC로 전달받아 로컬 SQLite의 `keyword_weights` 테이블 내 가중치 점수를 누적 업데이트하는 기능 구현.
6. **클라이언트 사이드 가중치 기반 정적 피드 재정렬 엔진 (Local Re-ranking Engine)**:
   - GCS CDN에서 다운로드받은 데일리 JSON 피드의 콘텐츠 카드 태그와 로컬 SQLite의 `keyword_weights` 점수를 조인/비교하여 우선순위 랭킹 스코어를 계산하고, 화면 렌더링 시 가중치 점수가 높은 카드를 최상단에 재배치하는 로컬 큐레이션 알고리즘 구현.
7. **StoreKit 2 온디바이스 결제 검증 모듈 (Mac App Store IAP 단일 경로)**:
   - **결제 경로 단일화 확정**: 결제는 **Mac App Store 인앱 결제(StoreKit 2 IAP)로 일원화**합니다. Gumroad/Stripe 등 외부 결제는 채택하지 않습니다(수수료·심사·구현 이원화 회피). 사용자가 Touch ID 등으로 $5.99 영구 인앱 결제를 실행하면 StoreKit이 거래(Transaction)를 발급합니다.
   - **[설계 변경] 영수증 검증은 Apple 프레임워크(Swift)에 위임**: Rust로 ASN.1 영수증 파싱 및 Apple Root CA 서명 체인을 직접 구현하는 방식은 고난도·고유지보수 부담이 크므로 채택하지 않습니다. 대신 **StoreKit 2가 제공하는 온디바이스 서명 검증(`Transaction.currentEntitlements`, `VerificationResult`)을 Swift 사이드(Tauri의 macOS 네이티브 플러그인/사이드카)에서 수행**하고, 검증된 엔타이틀먼트 결과(Boolean 라이선스 플래그)만 Tauri IPC로 Rust/React에 전달합니다.
   - **오프라인 라이선스 스위치**: StoreKit이 검증한 엔타이틀먼트 상태를 로컬에 캐시하여, 외부 인터넷 연결이 차단된 오프라인 환경에서도 3개 궤도 제한 등 PRO 기능을 즉각 해제하는 로컬 라이선스 스위치를 구축합니다.
8. **익명 UGC 수집용 서버리스 API 및 Cloudflare Workers 3단계 검증 파이프라인**:
   - 사용자가 동의(Opt-in)하고 실제로 "달성 완료" 처리된 커스텀 궤도 문장만 전송을 허용하는 클라이언트 보안 필터 구축.
   - Cloudflare Workers API를 설계하여, 사용자 IP 해싱 기반 Rate Limiting(1인당 1일 5회 초과 시 요청 거절) 적용.
   - UGC가 버퍼(`suggestions/raw_inputs.jsonl`)에 쓰인 뒤:
     1. **1단계(정규식)**: 비속어 및 URL/특수문자 자동 제거.
     2. **2단계(Gemini Vetting)**: Gemini Flash LLM이 문맥 내에 개인 정보(이름, 회사명 등)를 검사 및 제거하고, 해당 궤도가 주니어 트랙 혹은 시니어 트랙 중 어느 범주에 적합한지 자동으로 분석하여 레이블을 주입.
     3. **3단계(Human-in-the-Loop)**: 어드민 패널에 `Pending`으로 누적하고 관리자가 수동 승인(`Approve`)할 때 비로소 GCS의 트랙별 공통 추천 풀(`recommendations.json`)에 병합 배포.

### 6.4. 웹서비스 리팩토링에 따른 사이드 이펙트 최소화 및 방어 전략 (Side-Effects Mitigation Strategy)
기존 웹서비스의 활발한 실사용성을 훼손하지 않고, 데스크톱 앱을 위한 마이그레이션을 안전하게 수행하기 위해 발생 가능한 3대 사이드 이펙트와 구체적인 방어 메커니즘을 정의합니다.

1. **인증 경로 분기 및 데이터 형상 불일치에 따른 웹 클라이언트 동작 이상 방지**:
   - **[위험 정정] 실제 위험은 onSnapshot 누수가 아니라 인증 분기**: 현재 웹 프론트엔드는 Firestore `onSnapshot` 실시간 리스너를 사용하지 않으므로(인증 토큰 기반 REST 폴링 구조), 리스너 미해제 메모리 누수는 실질 위험이 아닙니다. 진짜 위험은 ① 웹 경로(`idToken` 인증 필수)와 데스크톱 경로(무인증)가 `usePaceNoteData` 훅에서 분기될 때, 토큰 미부착/만료 처리 누락으로 **웹 클라이언트의 인증 호출이 깨지는 것**, ② 유저별 Firestore 도큐먼트(웹)와 단일 사용자 로컬 SQLite(데스크톱)의 **응답 데이터 형상(Data Shape) 불일치**로 렌더링이 어긋나는 것입니다.
   - **대책**: `usePaceNoteData` 훅에서 환경 감지(웹/Tauri)를 단일 진입점으로 격리하고, 웹 경로는 기존 `fetchWithAuth`(토큰 주입·만료 재시도) 로직을 그대로 보존합니다. 두 경로의 반환값을 동일한 TypeScript 인터페이스(또는 런타임 검증 객체)로 정규화하여 형상을 완전히 일치시키고, 웹 회귀 테스트로 인증 호출 경로가 훼손되지 않았음을 검증합니다.
2. **콘텐츠 파이프라인 스키마 변경에 따른 구버전 클라이언트 크래시 방지**:
   - **위험**: 영어/일본어 어학 대신 수준별/트랙별 스키마가 확장되고 액션 챌린지(`action_challenge`)와 같은 신규 필드가 배포될 때, 정적 JSON 파일 구조의 물리적 필드명 변화로 인해 로컬 캐시를 가진 구버전 웹 프론트엔드가 이를 파싱하다가 `undefined` 참조 에러로 인해 화면이 하얗게 굳어버리는 크래시가 발생할 수 있습니다.
   - **대책**: **하위 호환성 가드(Graceful Fallback)**를 적용합니다. 프론트엔드 파싱부 초입에 단언(Assertion) 연산자 및 옵셔널 체이닝(`?.`)을 전면 적용하고, 기존 어학 영역 객체(`study` 등)가 부재하더라도 빈 구조로 무해하게 매핑되는 기본값 매퍼를 삽입하여 구버전 클라이언트의 정상 동작을 무조건 보장합니다.
3. **어드민 콘텐츠 GCS 동기화(`syncDailyGcs`) 시 동시성 충돌 및 Latency 증가 대응**:
   - **위험**: 관리자가 연속으로 글을 수정하거나 다중 어드민 세션이 동시에 데이터를 갱신할 때, GCS 파일 쓰기 경합(Race Condition)으로 최신 글이 누락되거나, 업로드 및 Cloudflare CDN 캐시 퍼지 처리의 동기식 지연으로 인해 어드민 API의 응답 속도가 대폭 저하(Latency 증가)될 수 있습니다.
   - **대책**: **비동기 작업 큐(Job Queue) 도입 및 낙관적 락(Optimistic Lock)** 방식을 적용합니다. `syncDailyGcs`를 API의 메인 요청 스레드에서 즉시 실행하지 않고, 메모리 내 인큐잉(Queueing) 혹은 Cloud Run의 Background Task로 위임하여 비동기 논블로킹(Non-blocking)으로 처리합니다. 또한 각 어드민 요청 시 Firestore 트랜잭션 버전을 대조하여 데이터 경합을 차단합니다.


## 7. 개발 및 출시 로드맵 (Roadmap)

### 7.1. 개발 기술 스택 제안 (100% 로컬리스 지향)
* **프레임워크**: **Tauri (Rust + React)**
  * *Tauri 선호 이유*: Electron 대비 앱 용량이 극단적으로 작고(10MB 수준), 백그라운드 서버 구동 없이 시스템 트레이(메뉴바) 제어, 로컬 암호화 DB 제어가 가장 효율적임.

* **동기화**: **Apple iCloud Drive** 파일 감지 동기화 (앱이 로컬 iCloud 앱 폴더에 데이터를 저장하면 macOS가 자동으로 기기간 동기화 처리)
* **AI API 클라이언트**: 클라이언트 앱 내부에서 직접 OpenAI/Gemini SDK를 활용해 사용자의 API Key로 호출 (중앙 프록시 서버 불필요).

## 8. 개인정보 보호 및 데이터 무수집 정책 (Privacy & Zero-Data-Collection Policy)

Pace Note는 사용자의 프라이버시를 최우선 가치로 삼으며, "기획단계부터 데이터 무수집(Privacy by Design)" 원칙을 고수합니다. 이는 사용자 신뢰도를 극대화할 뿐만 아니라, 개발사의 관리 책임을 최소화하여 고정 운영 비용 및 법적 리스크를 제로(0)로 만드는 핵심 기반입니다.

### 8.1. 데이터 무수집 원칙 (Zero-Data-Collection Principle)
* **서버 저장 원천 배제**: 사용자의 하루 할 일(Orbit), 주간 사색 및 회고록(Captain's Log), 태그 및 메모 등 일체의 기록은 개발사의 중앙 데이터베이스나 외부 서버로 **절대 전송되거나 저장되지 않습니다.**
* **완벽한 로컬 스토리지 상주**: 모든 사용자 데이터는 사용자의 맥북 로컬 파일시스템 내 안전하게 격리된 영역(Tauri 앱의 로컬 SQLite 및 마크다운 파일)에만 100% 로컬 저장됩니다.

### 8.2. 보안 연동, 동기화 및 백업 기술 정의
* **iCloud Ubiquity Container 기반의 자동 동기화 & 실시간 백업**:
  * **iCloud 전용 앱 컨테이너 활용**: Tauri 앱은 macOS가 부여하는 공식 iCloud Ubiquity Container 경로(`~/Library/Mobile Documents/iCloud~com~prisincera~pacenote`)를 감지하여 데이터베이스(SQLite) 파일 및 성찰 일지(마크다운)를 해당 폴더에 저장합니다.
  * **실시간 동기화 및 백업**: 파일이 해당 컨테이너에 쓰여지는 즉시 macOS의 iCloud 디먼(Daemon)이 이를 감지하여 애플의 보안 클라우드 서버로 실시간 동기화합니다. 사용자는 기기를 잃어버리거나 다른 맥북으로 전환하더라도 동일한 Apple ID로 로그인하는 즉시 어떠한 수동 복원 작업 없이 이전 기록이 100% 자동 복업 및 동기화됩니다.
  * **iCloud 자체 버전 관리 기능 활용**: iCloud Drive는 기본적으로 파일 버저닝(File Versioning)을 제공하므로, 회고 노트의 예기치 못한 유실이나 데이터 손상이 발생할 경우 사용자는 iCloud 시스템을 통해 과거 특정 시점의 데이터로 복원할 수 있습니다.
* **수동 오프라인 백업 및 외부 내보내기 (Manual Export/Import)**:
  * **휴대용 아카이브 생성**: 사용자가 직접 로컬 스토리지에 데이터를 소장하거나 타사 클라우드(Dropbox, Google Drive 등)로 수동 백업하고자 할 경우를 대비해, 앱 내에서 모든 기록 데이터를 단일 압축 파일(`.zip` 또는 `.json` 포맷)로 내보낼 수 있는 기능을 지원합니다.
  * **데이터 주권 보장**: 언제든 원하는 포맷(JSON, Markdown, CSV 등)으로 가공하여 내보낼 수 있어 사용자가 완벽한 데이터 소유권(Data Sovereignty)을 가집니다.
* **사용자 API Key (BYOK) 암호화 보관**:
  * 사용자가 입력한 Gemini 및 OpenAI API Key는 평문으로 노출되지 않으며, macOS의 시스템 보안 저장소인 **Keychain(키체인)**에 암호화 보관됩니다.
  * AI 호출은 프록시 서버를 경유하지 않고, 클라이언트 내부에서 직접 Google/OpenAI 공식 엔드포인트로 안전하게 다이렉트 호출(Direct HTTPS)됩니다.

### 8.3. 컴플라이언스 및 법적 리스크 제로화
* **글로벌 규제 프리(Free)**: 개인정보를 일체 수집 및 전송하지 않으므로, 유럽 GDPR, 미국의 CCPA/CPRA, 한국의 개인정보보호법 등 엄격한 글로벌 개인정보 규제 대상에서 완전히 제외됩니다.
* **컴플라이언스 비용 0원**: 개인정보 처리방침 고지 의무 및 DB 유출 위험 보상 보험 가입 등 물리적/법적 관리 소요가 발생하지 않아 유지 보수 및 운영 리스크가 최소화됩니다.

### 8.4. 로그인 프리 무중단 온보딩 및 기기간 연동 프로세스 (Login-Free Onboarding)
* **로그인 절차의 완전한 제거 (Zero-Friction)**:
  * 앱 최초 설치 및 실행 시 복잡한 이메일 가입, 소셜 로그인, 수동 애플 로그인 팝업 등을 전혀 요청하지 않습니다.
  * 실행 즉시 로컬 데이터베이스를 바탕으로 핵심 기능(오빗 생성, 회고록 기록)을 바로 사용할 수 있습니다.
* **OS 레벨 연동을 통한 투명한 동기화**:
  * 별도의 로그인 상호작용 없이 macOS 시스템에 설정된 애플 ID 및 iCloud Drive 활성화 여부를 자동 감지하여 동작합니다.
  * iCloud가 활성화된 상태라면 별도의 로그인이나 설정 조작 없이 기기간 동기화가 백그라운드에서 자동으로 투명하게 처리됩니다.
* **로컬 기반 라이선스 검증**:
  * PRO 버전 활성화 시에도 자체 계정이 필요 없으며, Mac App Store 구매 영수증(StoreKit 로컬 검증) 또는 오프라인 라이선스 키 1회 입력으로 기능을 즉시 잠금 해제합니다.

### 8.5. 데이터 무수집 기반의 추천 및 개인화 아키텍처 (Privacy-Respecting Orbit & Feed Personalization)
기존 웹 서비스의 중앙 집중식 서버 분석 기반 개인화(사용자 데이터를 중앙 DB로 올려 추천 알고리즘을 돌리는 방식) 대신, **클라이언트에 데이터를 격리한 상태에서 완벽한 개인 맞춤화를 실현하는 로컬 순환 랭킹 모델**을 채택합니다.

* **정적 CDN 기반의 공통 추천 풀 공급 (기본 모드 - $0 서버리스)**:
  * 개발사의 백엔드 파이프라인이 매일 새벽에 수준별 트랙 정적 JSON 피드(`daily/junior_${date}.json`, `daily/senior_${date}.json`) 및 공통 추천 풀(`recommendations.json`)을 GCS에 빌드 배포합니다.
  * 사용자의 앱은 구동 시 식별 정보(쿠키, 토큰 등)를 일체 싣지 않고 HTTP GET 요청을 보내 해당 정적 JSON 파일을 받아 로컬 SQLite에 다운로드합니다. 이 단계에서 어떠한 데이터 수집도 일어나지 않습니다.
* **온디바이스 가중치 랭킹 시스템 (Local Weight-Based Feed Re-ranking)**:
  * 사용자가 주말마다 작성하는 성찰 기록(`Captain's Log`)의 텍스트와 완료된 오빗 데이터는 오직 로컬 기기 내부에서 사용자의 API Key를 탑재한 로컬 AI 인프라를 통해 피드백 분석을 거칩니다.
  * AI 코치가 피드백 생성 과정에서 도출한 **'사용자 지식 및 실행의 병목/성장 키워드'**는 로컬 SQLite의 `keyword_weights` 테이블에 저장되며, 매일 새로운 데일리 테크 시그널 JSON 피드를 읽어올 때 해당 키워드를 내포한 콘텐츠 카드가 사용자 화면에 최우선적으로 정렬(Re-ranking)되도록 로컬에서 쿼리 필터링이 수행됩니다.
* **온디바이스 개인 맞춤형 AI 추천 (PRO / BYOK 모드)**:
  * 외부 서버에 일체의 완료 기록이나 로그를 올리지 않고 오직 로컬 SQLite에 적재된 지난 수개월 간의 오빗 완료 로그를 AI의 로컬 Context Window에 바로 주입(Keychain 기반 API 호출)함으로써, 100% 프라이버시가 안전하게 보호되는 초개인화 추천 궤도를 기기 안에서 직접 연동해 로컬 캐시에 적재합니다.

### 8.6. 익명화된 사용자 지정 궤도(UGC) 수집 및 피드백 루프 (Anonymized Crowdsourcing & Isolation Policy)
"개인 식별 정보의 수집 금지"가 "단순 텍스트 데이터의 생산성 배제"를 의미하지는 않습니다. 사용자들이 직접 생성하는 독창적이고 실용적인 커스텀 목표(UGC)는 서비스 가치를 발전시키는 훌륭한 자원입니다. 이를 **완벽한 익명성 및 인프라 비용 $0 원칙**, 그리고 **실시간 데이터 오염을 방지하는 격리 운영 정책** 하에 수집하는 구조를 설계합니다.

* **달성 완료된 궤도에 한정하여 기여 (UGC 품질 통제 정책)**:
  * 사용자가 단순히 목표를 *입력/생성한 시점*에는 데이터를 수집하지 않습니다. 무의미한 텍스트 기입(예: 오타, "asdf", 단순 테스트용 입력)으로 인한 데이터 오염을 방지하기 위함입니다.
  * 오직 사용자가 커스텀 목표를 **실제 실행하여 '달성 완료(Checked/Completed)' 처리한 성공 이력에 한해서만** 추천 기여 전송을 트리거합니다. 이는 문장의 현실성, 가치 및 실행 가능성이 1차로 사용자 본인에 의해 검증되었음을 의미합니다.
* **커뮤니티 추천 기여 (동의 기반 익명 수집 - Opt-in)**:
  * 커스텀 목표를 생성할 때 *"이 목표를 달성하면 다른 러너들과 익명으로 공유하여 추천 풀에 기여합니다"*라는 토글을 활성화(기본값: Off)한 경우에만 작동합니다.
  * 동의한 경우에 한해, 목표의 **[텍스트 내용]**과 **[카테고리]** 정보만 서버로 전송합니다. 이 과정에서 유저 ID, 이메일, IP 주소, 기기 식별값 등 **개인을 특정할 수 있는 일체의 메타데이터는 전송 단계에서 원천 차단**됩니다.
* **실시간 라이브 피드 반영 원천 금지 (Strict Data Isolation)**:
  * 악의적인 광고, 욕설, 스팸 유입이나 인프라 과부하로 인한 실시간 변동 문제를 방지하기 위해, 사용자가 입력한 커스텀 목표는 라이브 추천 풀(`recommendations.json`)에 **절대 실시간으로 직접 쓰여지거나 반영되지 않습니다.**
  * 사용자의 앱 역시 자신이 업로드한 추천 기여 항목을 실시간으로 서버에서 되읽어올 수 없습니다. 오직 쓰기(Write-Only)만 가능하도록 인프라 차원에서 차단합니다.
* **비동기 배치 윈도우 및 API Rate Limiting**:
  * **일일 배치 정제**: 전송된 데이터는 24시간 동안 완벽히 격리된 임시 버퍼 공간에만 쌓이며, 매일 새벽 딱 한 번 실행되는 백그라운드 파이프라인(`pacenote-composer.mjs`)에 의해서만 배치(Bulk)로 읽혀 정제 과정을 거칩니다.
  * **전송량 제한 (Rate Limiting)**: 스팸 전송 및 분산 서비스 거부(DDoS) 공격 방지를 위해 클라이언트 로컬 및 Cloudflare Workers 방화벽 수준에서 1인당 하루 최대 5회까지만 궤도 기여 전송이 가능하도록 요청 빈도를 제어합니다.
* **3단계 검증 필터링 (Three-Step Vetting Filter)**:
  1. **1단계: 규칙 및 키워드 필터링 (Regex/Blacklist)**: 텍스트 길이 검증(10자 이상 100자 이하), 특수기호 제거, 광고성 URL 및 사전에 정의된 비속어/욕설 블랙리스트 매칭을 통해 1차 기계적 제거를 수행합니다.
  2. **2단계: AI 기반 윤리 및 문맥 검증 (LLM Vetting)**: Gemini Flash AI 검수 가동해 문장 내에 개인정보(이름, 회사명 등)가 포함되어 있는지, 성인/도박/유해 정보가 있는지, 앱의 궤도 카테고리(Deep Work, Productivity 등) 목적에 부합하는 자연스러운 문장인지를 판별하고 정제합니다.
  3. **3단계: 관리자 최종 승인 검수 (Human-in-the-Loop)**: AI 필터까지 통과한 후보군을 어드민 대시보드 내 대기 상태(Status: Pending)로 적재합니다. 관리자가 어드민 패널에서 직접 눈으로 최종 확인하고 **"승인(Approve)" 버튼을 수동으로 누른 궤도들만 비로소 공통 추천 풀(`recommendations.json`)에 최종 병합**되어 배포됩니다. 이 단계는 AI의 오작동이나 우회 광고를 거르는 최종 안전판입니다.

## 9. 사업 검토 결론

본 개정 계획은 **"비용의 절대적 최소화"**와 **"지속 가능한 고수익 파이프라인의 안전한 안착"**에 방점을 두었습니다. 

중앙 서버나 유료 데이터베이스, 자체 AI API 게이트웨이를 완전히 배제하고, macOS 사용자들이 기본 소유한 **iCloud Drive**와 **로컬 SQLite**, 그리고 실피오에서 검증된 **사용자 API Key 연동(BYOK)** 방식을 채택하였습니다. 

이를 통해 연간 단 13만 원(애플 개발자 회비)의 고정비만으로 서비스를 영구 운영할 수 있는 구조를 확립했습니다. 이는 유저 수 증가에 따른 인프라 비용 부담이 전무하여, 런칭 시점부터 창출되는 매출의 99% 이상이 그대로 창업가의 순수익 파이프라인으로 정착되는 가장 이상적인 1인 비즈니스 모델이 될 것입니다.

## 10. Phase 0 — 웹 선행 개발 실행 계획 (Web-First Prerequisite)

데스크톱 앱(Tauri/Rust) 착수 전, **데이터 계약(JSON 스키마 + REST 응답 형상)을 웹에서 먼저 동결**하여 이식 리스크를 제거하는 선행 단계입니다. 본 단계는 100% Node/React 영역으로 Rust 학습 리스크가 없으며, 살아있는 웹 서비스를 훼손하지 않는 가산(Additive) 방식으로 진행합니다.

### 10.1. 최종 산출물 — Data Contract v2 (계약 동결)
> 📄 명세서 정본: [data_contract_v2.md](data_contract_v2.md) (Phase 0 산출물 1호)

* **피드 JSON 스키마 (가산 확장)**: 기존 `daily/${date}.json`(`signal` + `study`, **어학 웹 전용으로 보존**)을 유지한 채, 데스크톱용 `daily/junior_${date}.json` · `daily/senior_${date}.json` 을 신규 추가. 각 카드에 `action_challenge`(3 task) 객체 및 관심 도메인 메타, `schemaVersion` 필드 포함.
* **PaceNote 응답 형상**: `get_pacenote` / `toggle` / `accept` / `add` / `diary`의 입출력 JSON을 TypeScript 인터페이스로 명문화. 이 형상을 웹 REST(`/api/pacenote/*`)와 데스크톱 IPC가 공유.

### 10.2. 워크스트림
1. **A. 콘텐츠 파이프라인 추가 (`tech-composer.mjs` 신규)**
   - `study-composer.mjs`는 웹 전용으로 존치. 주니어/시니어 테크 시그널 생성 배치 `tech-composer.mjs` 신규 작성.
   - `action_challenge` 스키마 빌딩(Gemini Flash가 카드별 3 task 구조화 삽입).
   - `composer.mjs` Phase 5의 GCS 배포에 트랙별 JSON 가산 + GCS write 유틸 공통 모듈화.
2. **B. 어드민 실시간 동기화 (`syncDailyGcs` 신규)**
   - `admin-api.mjs`에 `syncDailyGcs(dateStr)` 신규 헬퍼(트랙 JSON + index 갱신 + Cloudflare 퍼지).
   - 비동기 Job Queue + 낙관적 락으로 동시성 충돌·Latency 방어(6.4.3).
   - 어드민 패널에 `action_challenge` 편집 UI 추가.
3. **C. 프론트 데이터 계층 추상화 + 트랙 UI**
   - `usePaceNoteData` 커스텀 훅 신설(웹=인증 REST / 데스크톱=IPC 분기 진입점), PaceNote 호출부 마이그레이션 + 응답 정규화 매퍼.
   - Daily Digest에 트랙 탭(주니어/시니어) + 관심사 필터 칩 추가(**어학 카드는 웹에 유지**).
   - `action_challenge` 카드 하단 [오빗 추가] 버튼 — "Click-to-Orbit" 웹 선구현(REST `/api/pacenote/add` 연동).
4. **D. 하위호환·사이드이펙트 가드 (라이브 서비스 보호)**
   - 피드 파싱부 옵셔널 체이닝/기본값 매퍼(6.4.2), `schemaVersion` 분기 안전판.
   - PaceNote 훅 분기 시 웹 인증 경로 회귀 테스트(6.4.1 정정 위험).

### 10.3. 실행 순서 (의존성 기준)
```
1) Data Contract v2 명세 동결 (선행)
2) A. tech-composer 추가 ──> 3) B. 어드민 syncDailyGcs
4) C. 프론트(훅+트랙UI+Click-to-Orbit)  [A 스키마 확정 후 병렬 가능]
5) D. 하위호환/회귀 테스트 ──> 6) 웹 스테이징 배포·검증 ──> 계약 확정 ──> (데스크톱 착수)
```

### 10.4. 비범위 (데스크톱 단계로 이월)
Tauri 셋업 · Rust 커맨드 · SQLite/IPC · Keychain BYOK · iCloud 동기화 · StoreKit IAP · 로컬 재랭킹 엔진. 본 Phase 0는 "계약 확정 + 라이브 웹 반영"까지만 수행합니다.
